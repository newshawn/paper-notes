## TL;DR

1. **核心结论**：提出 **EAPO (Entropy-Aware Policy Optimization)**——在 RLVR 设定下按 token entropy 调节 advantage；Qwen2.5-Math-7B 上 AIME24 Avg@32 **39.8%**（DAPO 35.7%，EntroAdv 36.8%），聚合 48.6% 为最强基线
2. **方法机制**：Four Quadrant Decomposition（polarity × entropy 2×2）诊断 token 学习信号 → 按 entropy 显式 re-weight advantage：`Â^CA_{i,t} = Â_{i,t} + α·sign(Â_{i,t})·detach(Â^H_{i,t})`
3. **为什么有效**：softmax policy gradient 数学上 `∇L_t ∝ (1-π)`，高熵 token 天然承载更多 reward 信息流；但 RLVR 均分 advantage 抵消这个偏好 → EAPO 显式恢复
4. 💡 **一句话精华**：Token 级 credit 不该均分——高熵 token（PHR 象限）才是 reasoning improvement 的主引擎，EAPO 用 entropy 显式把 advantage 推向它们。

## Method

### 核心思路

用 **Four Quadrant Decomposition** 诊断 RLVR 里 token 的学习信号：按 reward polarity (±) × entropy (high/low) 分 4 类：

- **PHR** (Positive High-entropy)：reinforce 不确定的成功选择
- **PLR** (Positive Low-entropy)：reinforce 自信的成功选择
- **NHR** (Negative High-entropy)：penalize 不确定的失败选择
- **NLR** (Negative Low-entropy)：penalize 自信的失败选择

**实证发现**：
- **PHR 是主引擎**——sustained 提升 accuracy 且 "generalizes beyond sampled rollouts"
- **PLR 低效**——达到相同 accuracy 需 **5.5× 更多 steps**，diversity 落后 12%
- NHR 比 NLR **1.5× 快**收敛，accuracy 更强
- 低熵象限早期有帮助但很快 saturate；**只有高熵象限能驱动长期 gain**

### 关键设计

**Entropy-Normalized Credit Score**：

$$\hat{A}^H_{i,t} = \text{clip}\!\left(\frac{H_{i,t} - \mu_H}{\sigma_H},\ -\frac{|\hat{A}_{i,t}|}{\phi},\ \frac{|\hat{A}_{i,t}|}{\phi}\right)$$

**Entropy-Aware Advantage**（替换 GRPO 原 advantage）：

$$\hat{A}^{CA}_{i,t} = \hat{A}_{i,t} + \alpha \cdot \text{sign}(\hat{A}_{i,t}) \cdot \text{detach}(\hat{A}^H_{i,t})$$

- $\alpha \in (0, 1]$ 控制强度（论文 α=0.2）
- $\phi = 2$ 限 entropy 贡献幅度
- `sign(Â)` 保证 entropy 校正**与 reward polarity 同向**（正奖励高熵 token 放大正 advantage；负奖励高熵 token 放大惩罚）
- `detach(·)` 让 entropy 项**不回传梯度**，避免 reward hacking

### 🧬 Delta from GRPO / DAPO / EntroAdv

- **vs GRPO / DAPO**：不改 rollout、不改 reward 来源、不改 KL；**只在 advantage 计算阶段对每个 token 按 entropy normalize 后 clip/scale**——极简叠加
- **vs EntroAdv (Cheng et al. 2025)**：EntroAdv 用 simple bonus；EAPO 用 clip + sign 的 principled 连续调制，基于 CMI 理论（Arumugam et al. 2021）——"credit a token can carry is upper-bounded by its entropy"
- **vs Forking (Wang et al. 2025c)**：Forking 用 binary mask 粗暴区分 high/low entropy；EAPO 是连续的

### 流程示例

解题："求 $\int_0^1 x^2 dx$"，policy 生成 tokens："$\int x^2 dx = x^3/3$, from 0 to 1 = **1/3**"。

按 Four Quadrant 分：
- "$\int$", "$x^2$", "$dx$"、"$= x^3/3$"：低熵（教材模板 / 积分常规），正奖励 → **PLR**（放大有限）
- "from 0 to 1"：**中熵**（多种表述都对）
- "**1/3**"：**高熵**（可能写 "1/3" / "0.333" / "$\frac{1}{3}$" / 甚至错算） → **PHR 激活**，EAPO 让它拿到大 |Ã|

**差异点**：DAPO 给所有 token 同 Ã；EAPO 给 "1/3" 这种高熵决策点放大信号。**高熵 token 正是 reasoning 发生的地方**——低熵只是模板补全。

### 🧩 因果链

- **问题**：RLVR 下所有 token 被均分 advantage，不区分贡献度
- **根因**：softmax policy gradient 数学上 `∇_{z_v} L_t ≈ Â_t·(1-π_{o_t})` 让高熵 token 承载更多信号——**但 GRPO 均分 advantage 抵消了这个偏好**
- **解法**：显式 entropy-aware re-weighting，让高熵 token 的 |Ã| 变大（按 entropy z-score clip）
- **效应**：Qwen2.5-Math-7B AIME24 Avg@32 +4.1（vs DAPO）；DeepScaleR-1.5B-8k AIME24 45.1%（best）；PHR 象限消融证明生成了"generalize beyond sampled rollouts"的能力

### ⚠️ What would break this

- **Open-ended 任务（无 verifiable reward）**（作者 Limitation 1）：subjective 场景（写作 / open dialogue）需要不同信号设计
- **> 7B 模型未验证**（Limitation 3）：四象限动态在超大规模是否成立未知
- **α / φ 跨域 tuning**（Limitation 2）：数学 domain 的 α=0.2, φ=2 迁到其他领域可能需重调
- **Entropy 估计噪声**（推测）：token 级 entropy 在训练前期 policy 差时可能是 noise 而非信号——论文未讨论前期稳定性

## Key Results

**训练配置**
- 基座：Qwen2.5-Math-7B、DeepScaleR-1.5B-8k（up to 24k tokens）
- 训练数据：DeepScaleR dataset
- 训练方法：EAPO (GRPO-based), α=0.2, φ=2, temperature=1.0, top-p=0.95
- 训练资源：论文未明说

**评测 Benchmark**（三问）

- **AIME24 / AIME25**：American Invitational Math Examination
  - 任务来源：美国数学奥赛官方题
  - 执行环境：离线生成 answer，数值精确匹配
  - 评测方式：Avg@32（32 次采样平均）+ Pass@32
- **AMC / MATH / Minerva / Olympiad**：数学竞赛，Avg@4 + Pass@4
- **MMLU-Pro**：通识知识测试（Appendix G）

**核心结果**

| Benchmark | EAPO (Qwen2.5-Math-7B) | DAPO | EntroAdv | KL-Cov |
|---|---|---|---|---|
| AIME24 Avg@32 | **39.8** | 35.7 | 36.8 | — |
| Aggregate | **48.6** | — | — | — |

DeepScaleR-1.5B-8k：AIME24 Avg@32 **45.1** (best)，aggregate **57.0**（EntroAdv 44.0, KL-Cov 43.9）

**意外发现**：
- PHR 单独训练产生 sustained 提升，PLR 单独训练需 **5.5× 更多 step** 达同样 accuracy
- **Accuracy 和 diversity 同时提升**——罕见，通常二者 trade-off

## Takeaway

### 对我研究的启示

结合本 wiki 的 Agentic RL Credit Assignment 方向：

1. **Credit assignment 不止"turn 粒度"**：token 粒度是被忽视的维度——现有 Wiki Credit-Assignment 讨论粒度时只到 token(GRPO) / turn / sequence IS，但"token 级 reward 分配"（不仅是 IS 粒度）是新切面
2. **Four Quadrant 分析法通用**：任何 "polarity × 其他维度" 的 2×2 都值得一拆——比如 turn-level credit 也可能有"正确 turn 的高熵 / 正确 turn 的低熵"的不同作用
3. **潜在结合点**：**turn-level credit + token-level entropy scaling**——turn 粒度 reward（IGPO / AT²PO）+ EAPO 的 token-level entropy 调节，二者**正交可合并**
4. **RLVR insight 能否迁到 agentic**：RLVR 有精确 verifier，agentic RL 通常没有；EAPO 的机制是否依赖 reward 的低噪声？开放题

### 🧠 理解核验

1. Four Quadrant 的 PHR / PLR / NHR / NLR 各代表什么 token？为什么 PHR 是 reasoning improvement 的主引擎，PLR 不是？
2. EAPO 公式 `Â^CA = Â + α·sign(Â)·detach(Â^H)` 里 `sign(Â)` 和 `detach(·)` 各自在防止什么？
3. softmax policy gradient 的 `∇ ∝ (1-π)` 和 CMI bound "credit ≤ entropy" 在数学上的联系是什么？

## Open Questions

- **Open-ended 场景**（作者 Limitation 1）：非 verifiable reward 下如何定义 PHR/PLR/NHR/NLR？
- **Scale laws**（Limitation 3）：四象限动态在 > 7B 成立吗？
- **α / φ 自适应**：训练过程中能否按 entropy 分布自动调？
- **结合方向**：EAPO × turn-level credit（IGPO / AT²PO）——turn 粒度 + token 粒度双层调制，目前无人做
