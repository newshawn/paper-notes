# ARPO: Agentic Reinforced Policy Optimization

- **ID**: 2507-arpo
- **Venue**: ICLR 2026
- **Link**: [alphaxiv](https://www.alphaxiv.org/abs/2507.19849) · [code](https://github.com/dongguanting/ARPO)
- **Tags**: #credit-assignment #entropy #branching #tool-use #no-reward-model

## TL;DR

**观察**：tool call 返回结果后，模型的前 10-50 个 token entropy 明显升高——说明这是关键但模糊的决策点。**做法**：高 entropy 时触发 **partial rollout branching**，从当前节点再采样 Z 条新路径，共享前缀 token 共享 advantage、分叉后 token 获得差异化 advantage。Tool-call 总数约为 GRPO 的一半，精度更高。

## Method

### 自适应 Branching

对每次 tool call 后监控 token entropy：

$$\Delta H_t = \text{Normalize}(H_t - H_{\text{initial}})$$

$$P_t = \alpha + \beta \cdot \Delta H_t$$

若 $P_t > \tau$，从当前节点 branch 出 $Z$ 条新 partial trajectory。

### Advantage Attribution

- **共享前缀 token**：由于 $r_{i,t}(\theta)$ 在共享段完全相等，GRPO 自动让它们的 advantage 贡献趋近组内均值
- **分叉后 token**：各自独立 advantage

GRPO 的对比：整条轨迹 reward 共享 → tool call 1 的 query 哪怕合理，也会被 tool call 2 的错误拖累。ARPO 避免了这种误伤。

### 流程示例（论文）

Q: "Finding Nemo 主角在 2020 年前被 USGS 记录在哪里？"

1. 初始 N=2 global rollout
2. T2 的 tool call 1 结果后 entropy 升高 → branch 2 条
3. 总共 4 条轨迹（T1、T2、B1、B2），共享段 token 共享 advantage，分叉段独立

## Key Results

- 3 领域、13 benchmark：数学（AIME、MATH）、知识密集（HotpotQA、WebWalker）、Deep Search（GAIA、HLE）
- 冷启动 SFT（54K Tool-Star 数据）→ RL
- Deep Search 只用 1K 样本就显著优于 baseline
- Qwen2.5-7B：ARPO 全程 tool-call 总数 ≈ GRPO 的 **50%**，精度反而更高

## Takeaway

1. **Tool call 后的 token entropy 是天然 branching trigger**——这个观察很干净
2. **共享前缀 + 分叉后** 的 advantage attribution 很聪明，本质是隐式的 importance sampling 粒度细化
3. **Tool-call 效率提升是 bonus**：只在关键点多采，省资源

## Open Questions
