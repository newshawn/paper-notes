# SALT: Step-level Advantage Assignment for Long-horizon Agents via Trajectory Graph

- **ID**: 2510-salt
- **Link**: [arxiv](https://arxiv.org/pdf/2510.20022)
- **Authors**: Jiazheng Li, Yawei Wang, David Yan, Yijun Tian, Zhichao Xu, Huan Song, Panpan Xu, Lin Lee Cheong
- **Tags**: #credit-assignment #trajectory-graph #long-horizon #plug-and-play

## TL;DR

把**同 prompt 下的多条轨迹**构造成一个 **graph**（不仅仅是树），在 graph 上量化每个 step 的质量并分配 advantage。Plug-and-play 兼容 GRPO 等 group-based RL，**不改 rollout、不加计算开销**。在 WebShop / ALFWorld / AppWorld 上一致提升。

## Method

### 核心思路（根据 abstract）

现有问题：outcome reward 下，"有益 action" 和 "有害 action" 在 multi-step 交互里**纠缠在一起**——整条轨迹共享一个 return，好坏动作的 credit 分不开。

SALT 的做法：
1. 对同一 prompt 的所有 rollout 构造 **trajectory graph**
2. Graph 上量化每个 step 的质量
3. 据此分配 step-level advantage

**关键特征**：
- 只靠 outcome reward 推导 step-level advantage，无需额外 RM 或 MC 估值
- 不修改 rollout 流程
- 无额外计算开销
- Plug-and-play（可叠加 GRPO）

### 与相邻工作的关系

| 方法 | 结构 | 信号源 |
|---|---|---|
| GiGPO [2505-gigpo] | 同状态聚类（隐式图） | Outcome return |
| TreeGRPO [2509-treegrpo] | 显式树 | Outcome return |
| SALT | **通用 graph**（含 DAG 合并） | Outcome return |

SALT 的 "graph" 比 tree 更一般：允许不同轨迹的**中间状态合并**（DAG），理论上能捕获更多的 credit 重叠。
