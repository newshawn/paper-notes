# AEPO: Agentic Entropy-Balanced Policy Optimization

## TL;DR

对 ARPO 的批判性继承：**过度依赖 entropy 信号导致训练崩溃**（单一链条过度分支 + 高熵 token 梯度被无差别裁剪）。AEPO 用两件套修正：
1. **Entropy Pre-Monitoring**：自适应分配全局 vs 分支采样预算 + 连续高熵施加 branching penalty
2. **Entropy-Balanced Policy Optimization**：保留高熵 token 的梯度 + 熵感知 advantage 估计

## Method

### 动机：两个相互耦合的病

1. **过度分支**：连续高熵时资源都集中到一个链条，缺乏 diversity
2. **高熵梯度消失**：GRPO 目标 $\mathcal{L} = \min(\delta \cdot \tilde{A}, \text{clip}(\delta, 1-\epsilon, 1+\epsilon) \cdot \tilde{A})$，当 $A>0$ 且 $\delta > 1+\epsilon$ 时，$\text{clip}$ 项变常数 → **梯度为 0**。高熵 token 的 $\delta$ 天然偏大（分布散），极易被裁 → 模型学不到探索行为

### 部件 1: Entropy Pre-Monitoring

先预生成 1 条完整轨迹，监控 $H_{\text{root}}$（问题初始熵）和 $H_{\text{tool}}^{\text{avg}}$（工具调用平均熵）。

**自适应全局采样数**：

$$m = k \cdot \sigma\big(\beta \cdot (H_{\text{root}} - H_{\text{tool}}^{\text{avg}})\big)$$

- $H_{\text{root}} > H_{\text{tool}}^{\text{avg}}$：问题本身方向模糊 → 增加 $m$，多采全局轨迹
- $H_{\text{root}} < H_{\text{tool}}^{\text{avg}}$：工具反馈不确定更高 → 减小 $m$，把预算倾斜到分支采样

### 部件 2: Branching Penalty

$$P_t = (\alpha + \gamma \cdot \Delta H_t)(1 - \hat{P}(l))$$

$l$ 为此前连续高熵分支的数量。$l$ 越长 → $P_t$ 越小 → 抑制过度分支。

### 部件 3: 高熵梯度保留

把 clip 项改写为：

$$\frac{1+\epsilon}{\text{sg}(\delta)} \cdot \delta \cdot \tilde{A}$$

sg = stop gradient。这样 clip 不再让梯度消失，只对更新幅度做约束。

## Key Results

- 基座：Qwen3-8B-Instruct / Qwen3-14B-Instruct / Llama3.1-8B / Qwen2.5-7B
- 数据：1K 开源 web search 样本（对齐 ARPO，无数据合成）
- 资源：16×H800，Bing Search API（US-EN）
- Benchmark：GAIA / HLE / WebWalkerQA / XBench-DR / FRAMES / HotpotQA / 2Wiki / MuSiQue / Bamboogle / GSM8K / MATH / AIME
- 全面优于 ARPO，尤其 deep information seeking 场景

## Takeaway

1. **"修正 ARPO"** 的结构清晰：一个部件治采样、一个部件治梯度——都围绕 entropy 的双刃剑性质
2. **sg(δ) 技巧值得记**：通用技巧，可以移植到其他 clipping 类目标
3. **Entropy Pre-Monitoring 是元级观察**：用一次轻量 rollout 决定如何采样，值得在其他场景借鉴
