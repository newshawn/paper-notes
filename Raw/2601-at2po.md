# AT²PO: Agentic Turn-level Tree Policy Optimization

- **ID**: 2601-at2po
- **Link**: [alphaxiv](https://www.alphaxiv.org/abs/2601.04767) · [code](https://github.com/zzfoutofspace/ATPO)
- **Tags**: #turn-level-is #tree-rollout #entropy #branching-penalty #credit-assignment

## TL;DR

三件套框架，将 agentic RL 的 rollout、reward、training 三个阶段**都对齐到 turn 粒度**：
1. **Entropy-Guided Tree Expansion**（替代 TreeGRPO 的随机挑选 + AEPO 的 branching penalty）
2. **Turn-wise Credit Assignment**（树结构 Monte Carlo 回传 per-turn value）
3. **ATPO**（turn-level IS ratio + asymmetric clipping，介于 GRPO token-level 和 GSPO sequence-level 之间）

在 7 个知识搜索 QA benchmark 上比 AEPO 最高 +1.84 pp，multi-hop 优势更显著。

💡 **一句话精华**：把 rollout / reward / training 三阶段统一到 turn 粒度——entropy-guided 树展开 + 熵加权 value 回传 + turn-level IS，完胜 token-level 和 sequence-level 两个极端。

## Method

### Phase 1: 初始化树
对 prompt 用 $\pi_\theta$ 采 $M=10$ 条独立 chain rollout，构建树。

### Phase 2: Entropy-Guided Expansion

计算每个非叶节点的 normalized entropy：

$$H_{\pi_\theta}(n) \approx \frac{1}{|y^k|} \sum_{y_t \in y^k} -\log \pi_\theta(y_t \mid x, y^{<k})$$

选 entropy Top-K=6 的节点 fork。引入**分支惩罚** $\alpha$ 防止同一节点被反复扩展。重复 $L=2$ 轮，最终 $10 + 2 \times 6 = 22$ leaves。

### Phase 3: Turn-wise Credit Assignment（递归回传）

$$V_n = \begin{cases} \hat{r}_n, & n \text{ is leaf} \\ \sum_{c \in C(n)} w_c V_c, & \text{otherwise} \end{cases}$$

$$w_c = \frac{H_{\pi_\theta}(c)}{\sum_{c'} H_{\pi_\theta}(c')}$$

每个 turn 的 advantage = $V_n$（直接作为 advantage，不做归一化）。

### Phase 4: ATPO 训练

**Turn-level IS ratio**：

$$s_{i,t}^{turn}(\theta) = \underbrace{\frac{\pi_\theta(y_{i,t} \mid \cdot)}{\text{sg}[\pi_\theta(y_{i,t} \mid \cdot)]}}_{\text{Part A: local token gradient}} \cdot \underbrace{\text{sg}\left[\left(\frac{\pi_\theta(y_i^{k(t)})}{\pi_{\theta_{old}}(y_i^{k(t)})}\right)^{\frac{1}{|y_i^{k(t)}|}}\right]}_{\text{Part B: turn-level geometric mean ratio}}$$

Part A 控制梯度流、Part B 作为常数控制幅度——相当于 GSPO 的 turn-level 版本。

**对比**：
| 方法 | IS granularity | 问题 |
|---|---|---|
| GRPO | token | 方差大、entropy collapse |
| GSPO | sequence | 粒度太粗，某 turn off-policy 全体受影响 |
| **ATPO** | **turn** | **精准 + 保留 fine-grained 信号** |

### 🧬 Delta from AEPO / GSPO

- **vs TreeGRPO**：随机挑节点 → **entropy Top-K 挑选** + branching penalty
- **vs AEPO**：flat partial branching → **显式树** + 熵加权 value 递归回传
- **vs GSPO**（sequence-level IS）：粒度从 sequence → **turn**（GSPO 对整条 ratio 共享 → 某 turn off-policy 拖累全体；ATPO turn-level 隔离）
- **vs GRPO**（token-level IS）：粒度从 token → turn（GRPO token 方差大导致 entropy collapse；ATPO turn 几何平均更稳）

### 🧩 因果链

- **问题**：multi-turn agentic RL 的 rollout / credit / optimization 三阶段粒度不对齐——token / sequence IS 都不合适
- **根因**：tool output 插入 → 轨迹天然分段，但现有方法按 flat token sequence 处理
- **解法**：**三阶段全 align 到 turn**——(1) rollout: entropy-guided 树展开；(2) credit: 熵加权 value 回传 per-turn；(3) optimization: turn-level IS ratio + Part A/B stop-gradient 技巧
- **效应**：Qwen3-4B multi-hop avg 48.81 vs AEPO 46.95 (+1.84 pp)

### ⚠️ What would break this

- **$V$ 直接作 advantage 未归一化**：训练稳定性在更大 M / 更深 L 下是否崩？论文未系统消融
- **熵双重放大偏差**：entropy 同时用于树展开 + value 聚合，若某 token entropy 估计偏高，会在 rollout 和 credit 两阶段复合放大
- **"Turn" 对非 QA 任务可能不清晰**：GUI agent / embodied 任务没有自然 turn 边界，需额外定义 (推测)

## Key Results

- 基座：Qwen3-4B / Qwen3-8B / Qwen2.5-7B
- Benchmark：7 个知识搜索 QA（multi-hop + single-hop）
- Qwen3-4B multi-hop avg：**48.81** vs AEPO 46.95

## Takeaway

1. **"Turn" 是正确的 IS 粒度**——token 方差大、sequence 太粗的中间态
2. **三个阶段对齐到同一粒度**是强原则：采样、reward、优化都 turn-level
3. **熵加权的 value 递归回传** 是个灵活机制——比均值聚合保留更多探索信号

### 🧠 理解核验

1. Turn-level IS 为什么同时比 token-level（GRPO）和 sequence-level（GSPO）都好？推导其 variance 分析
2. Part A / Part B 的 stop-gradient 分工是什么？为什么 Part B 要作为常数？
3. 熵加权的 value 聚合 $w_c = H(c) / \sum H(c')$ 的直觉含义是什么？跟均值聚合的差别在哪？

## Open Questions

- Entropy 同时用于树展开 + value 聚合，会不会双重放大某种偏差？
- 树深 $L$ 如何随任务长度自适应？
